# TripAnchor

A Chrome extension that lets you collect places (events, restaurants, hotels, attractions) from any webpage while you plan a trip, then export them all into a **Google My Map** so you can see every place on a single map and check distances/transit between them.

No paid APIs, no accounts, no servers. All data lives in your browser.

## Why

Planning a trip usually means juggling 15 tabs across Eventbrite, TripAdvisor, OpenTable, blogs, etc. There is no easy way to gather every "place I'm considering" onto one map so you can eyeball distances and plan transit. TripAnchor fixes that.

## Install (developer mode)

1. Clone or download this repo.
2. Open `chrome://extensions/` in Chrome (or any Chromium browser: Edge, Brave, Arc).
3. Toggle **Developer mode** on (top right).
4. Click **Load unpacked** and pick the `TripAnchor/` folder.
5. Pin the TripAnchor icon to the toolbar.

> Icons are generated by `python3 scripts/make_icons.py` (requires Pillow). They ship pre-built so you usually don't need to run it.

## Daily use

1. Open a page with an event/restaurant/venue (Eventbrite, OpenTable, TripAdvisor, an Airbnb experience, etc.).
2. Click the TripAnchor icon. The popup auto-detects candidate places from:
   - `<script type="application/ld+json">` schema.org Event/Place/Restaurant/Hotel/Attraction blocks.
   - OpenGraph / `geo.position` / `ICBM` meta tags.
   - Embedded Google Maps iframes (extracts coordinates from the URL).
3. Tick the candidates you want and click **Add selected**.
4. If auto-detection misses the address (some pages render late or hide structured data, e.g. Google Search results, Wikipedia, blog posts), **select the place name** on the page, right-click, and choose **Add selection to TripAnchor (resolve via Google Maps)**. TripAnchor opens a new Google Maps tab, pre-fills its search with your selection, and shows a small confirmation banner. Click the right result in Maps, then press **Save this place** in the banner — TripAnchor captures the real coordinates from the Maps URL and stores them alongside the original source page.
5. To capture a whole page (e.g. an Airbnb listing where the URL itself is the reference), right-click the page background and pick **Add this page to TripAnchor**.
6. Repeat across as many tabs as you like. The popup shows everything saved on the current trip.
7. When ready, click **See my trip**.

You can manage multiple trips, rename them, switch between them, and back up everything to a JSON file from the options page (the **Manage** link in the popup header).

## See your trip on one map

Click **See my trip** in the popup. TripAnchor will:

1. Open <https://mymaps.google.com/> in a new tab.
2. Create a fresh map.
3. Import your mappable saved places from an in-memory CSV.
4. Select the location and marker-title columns and leave you on the populated map.

If Google asks you to sign in, finish signing in and TripAnchor will resume when
you return to My Maps. If Google's interface has changed or is not in English,
the TripAnchor panel tells you to download and import a CSV/KML file manually.

Saved items without coordinates or an address cannot appear on a map. TripAnchor
skips them during automatic import and reports how many were skipped.

If you'd rather grab the file without opening My Maps, open **Download manually** in the popup and use **Export CSV** / **Export KML** directly.

### CSV vs KML

- **CSV** — every saved place is included. Places without coordinates are geocoded by My Maps from the `Address` column on import.
- **KML** — only places with real coordinates are emitted, so the result is exact but smaller. Use this when you've resolved every place via the "Add selection" Maps confirmation flow.

## Privacy

- Trip data is stored in `chrome.storage.local` on your machine. When you click **See my trip**, TripAnchor uploads a generated CSV of the mappable saved places into the opened Google My Maps tab.
- The extension only scans the active tab when you click the icon or use the right-click menu. During **See my trip**, it temporarily injects the My Maps importer into the opened Google My Maps tab. There are no persistent content scripts.
- Backups (JSON files) you create are entirely under your control.

## Limitations

- Google does not provide a public API to write into your Google Maps Saved Lists, so TripAnchor exports to Google **My Maps** (a separate, importable map product). It's a one-time file import per trip rather than a live sync.
- The one-click My Maps import automates Google's English web interface. Google can change that interface without notice, so TripAnchor falls back to a manual CSV download when it cannot continue reliably.
- Heavy JS-rendered pages might not expose JSON-LD until after load; if so, scroll/wait a beat and click **Rescan** in the popup.
- `chrome.storage.local` is per-browser-profile and is not synced across devices. Use **Download backup** in the options page if you want to move data.

## Development

There is no build step — everything is plain ES modules and classic scripts. Edits hot-reload by clicking the refresh icon on `chrome://extensions/`.

To regenerate icons:

```bash
pip install pillow
python3 scripts/make_icons.py
```
